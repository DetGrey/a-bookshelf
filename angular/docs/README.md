## What to read first
1. Start with the blueprint for screens, flows, and data contracts.
2. Use the style guide to match the visual system.
3. Read the Angular architecture doc before writing any Angular code.
4. Follow the implementation plan to rebuild in phases.
5. Use the checklist as a final QA pass.

## Platform note
The blueprint, implementation plan, style guide, and checklist are framework-neutral. The Angular architecture doc is Angular-specific and supersedes any structural decisions in the other docs.
