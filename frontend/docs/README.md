## What to read first
1. Start with the blueprint for screens, flows, and data contracts.
2. Use the style guide to match the visual system.
3. Follow the implementation plan to rebuild in phases.
4. Use the checklist as a final QA pass.

## Platform note
These docs are written to be framework-neutral. They can guide a rebuild in web, mobile, desktop, or another UI stack by mapping the concepts to local app patterns.
